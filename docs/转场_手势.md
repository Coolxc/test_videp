# 优化方案1：转场、绘画顺序、图片 Prompt 质量

## 问题 1：转场生硬 — 改为手擦除刷新

### 1-1. 现状

当前 `PaperPullTransition.tsx` 实现的是**白纸从上向下滑入**覆盖旧画面，效果像翻页，但：
- 没有手参与转场，与"手绘"主题脱节
- 从上往下滑入的动效比较生硬，不够流畅
- 转场时长仅 15 帧（0.5 秒），太快

### 1-2. 目标效果

YouTube 白板视频标准转场：**一只手（或擦板）从左向右擦过画面，旧内容被擦除，露出干净白板**。

```
帧 1:                       帧 N/2:                     帧 N:
┌──────────────────┐       ┌──────────────────┐       ┌──────────────────┐
│ ╭─╮              │       │        ┃ ╭─╮     │       │                  │
│ │旧│  旧内容      │       │ 白板 ✋┃ │旧│    │       │   干净白板        │
│ ╰─╯              │       │        ┃ ╰─╯     │       │   （等待新场景）   │
└──────────────────┘       └──────────────────┘       └──────────────────┘
                            手从左向右擦
```

### 1-3. 实现方案

**新建组件** `HandWipeTransition.tsx`，替换 `PaperPullTransition`。

核心结构：
```
┌─ AbsoluteFill (zIndex: 50) ─────────────────────┐
│                                                   │
│  ┌─ 白色遮罩 div ──────────┐                      │
│  │ width = 1920 * progress  │   ← 从左向右扩展    │
│  │ height = 1080            │                      │
│  │ backgroundColor: #FFF    │                      │
│  └──────────────────────────┘                      │
│                              ✋ ← 手图片在遮罩右边缘│
│                                                   │
└───────────────────────────────────────────────────┘
```

**关键参数**：
- 手图片：复用 `writing-hand-small.png`（掌心朝向画面，像在擦白板）
- 手的位置：始终在白色遮罩的右边缘，随遮罩扩展从左移到右
- 手的旋转：约 90° 旋转，让手掌面朝右侧（像在擦白板）
- 缓动：ease-in-out（开始慢 → 中间快 → 结束慢）
- 转场时长：建议增加到 20-25 帧（0.7-0.8 秒）

**代码核心逻辑**：
```typescript
const HandWipeTransition: React.FC<Props> = ({ startFrame, durationFrames }) => {
  const frame = useCurrentFrame();
  const progress = clamp((frame - startFrame) / durationFrames, 0, 1);
  if (progress <= 0) return null;

  // ease-in-out 缓动
  const eased = progress < 0.5
    ? 2 * progress * progress
    : 1 - Math.pow(-2 * progress + 2, 2) / 2;

  const wipeX = CANVAS_W * eased;  // 白色遮罩右边缘 x 坐标

  return (
    <AbsoluteFill style={{ zIndex: 50 }}>
      {/* 白色遮罩：从左向右扩展 */}
      <div style={{
        position: "absolute",
        left: 0, top: 0,
        width: wipeX, height: CANVAS_H,
        backgroundColor: "#FFFFFF",
      }} />
      {/* 手：在遮罩右边缘 */}
      <Img
        src={staticFile("assets/writing-hand-small.png")}
        style={{
          position: "absolute",
          left: wipeX - 80,    // 手掌在遮罩边缘
          top: CANVAS_H / 2 - 200,  // 垂直居中
          width: 280, height: 385,
          transform: "rotate(90deg)",  // 手掌朝右，像在擦
          zIndex: 51,
        }}
      />
    </AbsoluteFill>
  );
};
```

**改动范围**：
| 文件 | 改动 |
|------|------|
| `remotion-project/src/HandWipeTransition.tsx` | 新建（~60 行） |
| `remotion-project/src/WhiteboardVideo.tsx` | 替换 PaperPullTransition 引用（2 行） |
| `remotion-project/src/WhiteboardVideo.tsx` | 转场时长调整（可选） |

**工作量**：小（约 1-2 小时）

## 问题 2：绘画顺序缺乏真实感

### 2-1. 现状

所有元素内部的路径都是**从上到下**的顺序绘制，因为 `_extract_branches` 中 numpy 按行扫描输出骨架像素，分支的排列自然是从 y 小到 y 大。

效果：金字塔从顶点开始一行行往下画，像打印机。不像人画画——人会先画金字塔的三角形外框，再画内部横线和图标。

### 2-2. "先画框再填充"的真实绘画逻辑

真人在白板上画一个元素时的典型顺序：
1. **先画外轮廓/主结构线**（长线条 → 确定形体）
2. **再画内部细节**（短线条 → 补充信息）
3. **最后画装饰细节**（点缀、阴影笔触 → 增加质感）

### 2-3. 这改动大不大？

**不大。** 只需在 Python 路径提取脚本中，对每个元素内的路径按**长度降序排列**即可。

原理：
- 外轮廓/框架线 = 骨架中最长的分支（如金字塔的三条斜边、脑的外圈曲线）
- 内部细节 = 较短的分支（如金字塔内横线、脑沟纹理）
- 装饰笔触 = 最短的分支（如汗珠、阴影线）

长度降序排列后，绘画顺序自然变成：**框架 → 结构 → 细节**。

### 2-4. 实现

**文件**: `scripts/extract_drawing_paths.py`

在 `_assign_paths_to_elements()` 中，对每个元素的路径按长度降序排列：

```python
# 当前代码：按元素分组输出，元素内保持位置顺序（从上到下）
result = []
for elem in elements:
    eid = elem["id"]
    for i, rp in enumerate(raw_paths):
        if path_elem_map[i] == eid:
            result.append({"d": rp["d"], "elementId": eid})

# 修改为：按元素分组，元素内按路径长度降序（长线条先画）
result = []
for elem in elements:
    eid = elem["id"]
    elem_paths = [(i, rp) for i, rp in enumerate(raw_paths) if path_elem_map[i] == eid]
    # 按路径长度降序：长线条（外轮廓）先画，短线条（细节）后画
    elem_paths.sort(key=lambda x: x[1]["length"], reverse=True)
    for i, rp in elem_paths:
        result.append({"d": rp["d"], "elementId": eid})
```

**改动范围**：3-5 行代码

**效果对比**：

```
修改前（从上到下）:              修改后（长线条优先）:

金字塔绘画顺序:                 金字塔绘画顺序:
1. 顶点一小段                   1. 左斜边（最长线）  ╲
2. 顶点附近一小段               2. 右斜边           ╱
3. 灯泡顶部                    3. 底边            ───
4. 灯泡圆弧                    4. 中间横线         ──
5. ...（从上往下扫描）          5. 上部横线          ─
6. 底边（最后才画底部）         6. 灯泡轮廓        ○
                               7. 文件夹轮廓      □
                               8. 火柴人          ╈
                               9. 细节笔触...

像打印机                       像画师
```

**工作量**：极小（10 分钟）

## 问题 3：图片生成 Prompt 质量

### 3-1. YouTube 高赞手绘视频的图片有什么特征

研究大量 YouTube 高赞白板手绘频道（Kurzgesagt 简笔画版、TED-Ed 白板系列、各类知识博主白板视频）后，总结出优质白板手绘图片的核心特征：

**A. 视觉极简主义**
- 每个概念用**一个标志性符号**表达，不画多余细节
- 时间 = 时钟，不画出每个刻度和数字
- 金钱 = 带 $ 符号的钞票图标，不画精细的纸币纹理
- 人 = 火柴人/简笔人，不画五官细节
- 原则：**观众用 0.5 秒就能辨识出画的是什么**

**B. 线条是唯一的表现手段**
- **只用轮廓线**，不填充、不上色、不画阴影
- 线条粗细一致（2-4px），不追求压感变化
- 线条干净利落，**没有交叉阴影（cross-hatching）、没有涂抹、没有渐变**
- 这一点当前 prompt 做得不够——scene2 的脑部图有大量交叉阴影线，导致骨架提取出数百条碎片路径

**C. 元素的"岛屿"布局**
- 每个元素周围有**大量留白**（元素间距 ≥ 元素自身宽度的 50%）
- 元素之间**绝不接触、不重叠**
- 这对路径提取和元素分组至关重要——重叠的元素会被识别为同一个骨架连通分量

**D. 强语义图标，不是写实插画**
- 不画"一个详细的人脑解剖图"，画"一个脑形轮廓里面三个灯泡"
- 不画"一个人坐在办公桌前"，画"一个火柴人 + 一个方块电脑 + 波浪线表示打字"
- 用**箭头、对比框、流程图**来表达关系，不用叙事性构图

**E. 适合矢量化和路径提取的技术特性**
- 纯黑线条 on 纯白背景（#000 on #FFF），无中间灰度
- 线条闭合或明确终止，不留开放端
- 元素内部线条简单（5-15 笔画），骨架提取后每个元素产生 10-30 条路径（而非数百条碎片）

### 3-2. 当前 Prompt 的问题

用上述标准审视当前的三张图片：

| 场景 | 当前效果 | 问题 |
|------|---------|------|
| scene1 (人+钟+钱) | 较好：火柴人、闹钟、纸币都是简笔画 | 闹钟的刻度线太密，产生碎片路径 |
| scene2 (脑) | **差**：脑部有大量交叉阴影线 | 交叉阴影产生 593 条碎片路径，手绘效果像在涂鸦而非画画 |
| scene3 (金字塔) | 较好但偏复杂：线条有多笔重叠描边 | 重叠描边导致骨架分支多，389 条路径 |

**核心问题**：当前 prompt 没有强调"只用轮廓线、不画阴影填充"，导致生图模型添加了交叉阴影和重叠描边，这些对最终视频的手绘效果有害。

### 3-3. Prompt 优化方案

**修改文件**: `scripts/generate_prompts.py`

需要修改两处 system prompt（风格指南 + 场景 prompt），在画风规则中**强化轮廓线约束**，**明确禁止阴影和填充**。

#### A. 关键新增/修改的 prompt 规则

当前 prompt 已有的好规则（保留）：
- 纯白背景 #FFFFFF
- 不出现文字
- 元素间留白
- 简笔画风格

**需要新增的规则**：

```
1. 线条规则 — 只用轮廓线
   - 每个元素只画外轮廓，不画内部填充
   - 绝对禁止：交叉阴影(cross-hatching)、斜线填充、涂抹阴影、灰度渐变
   - 绝对禁止：同一条线画多笔重叠描边（每条线只画一笔）
   - 线条粗细均匀一致（约 3px），不追求压感变化
   - 结果应像白板马克笔一笔画出的线条

2. 元素复杂度限制
   - 每个元素最多 10 笔画，用最少的线条传达语义
   - 人 = 圆形头 + 直线身体 + 直线四肢（火柴人，共 6 笔）
   - 脑 = 一个云朵形轮廓线（共 1 笔连续曲线）
   - 时钟 = 圆 + 两条指针线（共 3 笔）
   - 金字塔 = 三角形 + 两条横线（共 5 笔）
   
3. 视觉隐喻优先
   - 不画写实物体，画概念符号
   - 用图标式表达：💡→ 圆+线段, 🧠→ 云朵轮廓, ⏰→ 圆+指针
   - 用箭头和简单几何形状表达关系
```

# 修复方案2：手拿笔尺寸与定位 — 对标 YouTube 白板视频效果

## 一、问题现象

手和笔已经能正确显示和跟随线条移动了，但**尺寸特别小**，不像真实的白板视频效果。

YouTube 上的白板手绘视频（VideoScribe / Doodly 风格）中，手拿笔是画面中一个醒目的视觉元素，占画面宽度的 15-20%。当前的手只有 120×120px，在 1920×1080 画面中仅占 6.25%，视觉上像一个小图标在移动。

## 二、根因分析

### 2-1. 手的图片被过度缩小

**原始图片**：`whiteboard-animation/assets/drawing-hand.png`，872×1200px，高清 RGBA

**部署脚本** `deploy_resources.py` 第 99 行：
```python
scale = 200 / max(h, w)  # max_dim=200 → 缩到 145×200
```

缩小到最大边 200px 后存为 `writing-hand-small.png`。

### 2-2. 渲染尺寸过小且宽高比错误

**MaskRevealAnimation.tsx 第 184-185 行**：
```typescript
width: 120,    // 120px，仅占画面 6.25%
height: 120,   // 强制正方形，原图是竖版 (0.73:1)，手被压扁
```

### 2-3. 笔尖偏移和旋转中心不正确

**第 182-188 行**：
```typescript
left: point.x - 10,          // 笔尖偏移仅 10px（手放大后偏移应成比例增大）
top: point.y - 30,           // 笔尖偏移仅 30px
transformOrigin: "10px 30px", // 旋转中心不在笔尖位置
```

### 2-4. 对比 YouTube 白板视频标准

| 维度 | 当前值 | YouTube 标准（VideoScribe/Doodly） |
|------|--------|----------------------------------|
| 手占画面宽度 | 6.25%（120/1920） | 15-20%（~300-400px） |
| 宽高比 | 1:1（正方形，变形） | 原始比例 0.73:1（竖版） |
| 图片分辨率 | 145×200（模糊） | ≥350×480（清晰） |
| 笔尖定位 | 偏移 (10,30)，不准 | 笔尖精确对准绘画点 |
| 旋转中心 | (10,30)，不在笔尖 | 以笔尖为轴旋转 |

## 三、修复方案

### 3-1. 提高部署图片分辨率

**文件**: `scripts/deploy_resources.py`

**第 99 行**，max_dim 200 → 600：
```python
# 修复前
scale = 200 / max(h, w)

# 修复后
scale = 600 / max(h, w)
```

效果：原图 872×1200 → 部署后 436×600（清晰度满足 350px 宽渲染需求）

### 3-2. 修正渲染尺寸、笔尖定位、旋转中心

**文件**: `remotion-project/src/MaskRevealAnimation.tsx`

**分析原始手图的笔尖位置**：

```
原图 872×1200：
  ┌──────────────────┐
  │  ✎ ← 笔尖 (150,55) │
  │    ╲              │
  │     ╲ 笔身        │
  │      ╲            │
  │       手          │
  │                   │
  └──────────────────┘

笔尖位置比例：x = 150/872 ≈ 17%, y = 55/1200 ≈ 4.6%
```

**渲染参数计算**：

```
渲染宽度 = 350px（350/1920 = 18.2%，符合 YouTube 标准）
渲染高度 = 350 × (1200/872) = 481px（保持原始宽高比）

笔尖在渲染图中的位置：
  penTipX = 350 × 0.17 = 60px
  penTipY = 481 × 0.046 = 22px
```

**第 177-192 行**，DrawingHand 的 return 改为：

```typescript
return (

    src={staticFile("assets/writing-hand-small.png")}
    style={{
      position: "absolute",
      left: point.x - 60,              // 笔尖 X 偏移（修复前: -10）

      width: 350,                       // 手宽度（修复前: 120）
```

### 3-3. 可选优化：柔和旋转

当前旋转直接使用路径方向角度，路径方向变化大时手会剧烈旋转。YouTube 白板视频中手的旋转非常柔和（通常 ±20° 以内）。

可选方案 A — 缩小旋转幅度：
```typescript
// 当前
transform: `rotate(${smoothedAngle}deg)`

// 优化：只取 30% 的角度变化，手不会剧烈旋转
transform: `rotate(${smoothedAngle * 0.3}deg)`
```

可选方案 B — 限制旋转范围：
```typescript
const clampedAngle = clamp(smoothedAngle, -25, 25);
transform: `rotate(${clampedAngle}deg)`
```

**建议先用方案 A（`* 0.3`）**，效果最接近 YouTube 白板视频的自然感。
