# 封面提示词模板（文生图专用）

本视频的封面风格：**温暖米黄色扁平矢量插画**，**9:16 竖版构图**（与视频 1080×1920 比例一致，无裁切）。

## 生成方式（二选一）

### ✅ 首选：Agent 内置 image_gen

如果当前 Agent 环境（WorkBuddy / Claude Code / Cursor 等）提供了 `image_gen` 工具，**直接调用生成**：

```
image_gen(
  prompt=<按下方模板构造的中文提示词>,
  size="1024x1536",                # 9:16 竖版，与视频比例一致
  output_dir="{OUTPUT_DIR}",
  quality="high",
  n=1
)
```

生成后检查文件名并重命名为 `{OUTPUT_DIR}/cover.png`。

### ⚠️ 降级：外部文生图

如果当前环境没有 image_gen 工具：
1. 把下面完整模板输出到对话里给用户
2. 让用户粘贴到 nano banana / Midjourney / DALL·E 等外部工具
3. 用户生成后保存为 `{OUTPUT_DIR}/cover.png`

> **提示词全程中文，不输出英文。** 模板中 Title Block / Subtitle Tag / Tail Style Block 三段必须逐字复制，不要改写——文生图模型对措辞变化极敏感，改一个字风格就跑偏。

## 完整模板（直接使用）

```
一幅扁平矢量插画风格的技术图解，温暖的米黄色纯色背景，9:16 竖版构图。

顶部居中：黑色粗体标题，中大号（32pt），手写中文字体，中等粗细笔触，字不要太大，整洁紧凑，标题文字为"[视频标题中文]"。
标题下方带有英文副标题，常规粗细（16pt），灰色手写体，副标题文字为"[视频英文副标题]"。

[主要视觉描述：图解展示什么内容，包含哪些插画图标]。
[布局描述：各元素如何排列，使用什么形状和连接方式；竖版画面从上到下组织信息]。
[色彩描述：哪些元素使用什么颜色，关键文字如何高亮]。

温暖的米黄色（#faf5eb）纯色背景，无网格线。
扁平矢量插画风格，非简笔画，色彩饱和但柔和，
暖橙色（#e8943a）和深青绿色（#3a9e8e）为主色调，
红色（#e74c3c）用于警告和强调，
手写体中文标注文字，角落点缀少量浅绿色和浅橙色装饰圆点，
充足留白，干净温暖的演示幻灯片美学。
9:16 竖版构图，高分辨率。
--ar 9:16
```

## 要替换的占位符

| 占位符 | 含义 | 示例 |
|--------|------|------|
| `[视频标题中文]` | scene-config.json 的 `meta.title`，或主标题 | `"LightClawACE 架构拆解"` |
| `[视频英文副标题]` | 配合中文的英文副标题，一般是 topic 相关的英文 | `"LightClawACE Architecture"` |
| `[主要视觉描述]` | 一段话描述画面核心视觉元素（有什么插画、图标、概念隐喻） | `"中心位置展示一个大脑剖面图，四周分布着6个小图标（齿轮、手、眼睛、盾牌、书本、数据流），每个图标用细线连接到中心大脑"` |
| `[布局描述]` | 元素如何排布（中心辐射/左右对照/上下递进/流程链条） | `"中心-辐射布局，大脑居中偏上，6个图标等距分布在下方半圆弧上"` |
| `[色彩描述]` | 哪些元素用什么颜色，哪些文字高亮 | `"大脑使用暖橙色主色，6个图标用深青绿色线条，关键文字\"智能体\"用红色高亮"` |

## 设计规范（给 AI 写描述时参考，不要直接写进提示词）

### 色板
- 背景 `#faf5eb`（温暖米黄）
- 暖橙 `#e8943a`（主色，重点插画/高亮）
- 深青绿 `#3a9e8e`（次色，次要插画/标签）
- 珊瑚红 `#e74c3c`（警告/强调/关键概念）
- 柔蓝 `#4a90d9`（数据可视化元素）
- 黑/深灰（标题和正文）
- 白（卡片/框填充）

### 插画风格
- **扁平矢量插画**（非简笔画、非线条涂鸦）
- 图标是彩色的、风格化的、精致的
- 像微信公众号技术文章封面那种质感

### 构图规则
- **必须 9:16 竖版**（与视频比例一致，Cover 组件无裁切）
- 留白充足（主内容占画布 60-75%）
- 数字列表用 ①②③
- 分类标签用「」中文方角括号

## 常用布局模板

当 AI 要写"主要视觉描述 + 布局描述"时，可参考以下范式：

### 1. Hub-and-Spoke（中心辐射）
适合：一个核心概念 + N 个相关要素。
```
[主要视觉描述]：中心位置一个大[核心图标]（如大脑/芯片/服务器），
周围等距分布 N 个小图标（图标1描述、图标2描述、...），
每个小图标通过细线连接到中心。

[布局描述]：圆形辐射布局，中心图标比周围图标大 2 倍，
连线用深青绿色，线条末端有小圆点装饰。
```

### 2. Process Flow（流程链）
适合：顺序/步骤/管线。
```
[主要视觉描述]：从左到右一条水平流程线，上面依次排列 4 个步骤卡片，
每个卡片包含一个插画图标和简短中文标签，
卡片之间用箭头连接，整体像一条生产线。

[布局描述]：水平线性布局，4 个圆角方形卡片等距排列，
箭头用暖橙色，每个卡片下方有步骤编号 ①②③④。
```

### 3. Multi-Column Comparison（并列对比）
适合：A vs B、新旧对比、方案对照。
```
[主要视觉描述]：画面分为左右两栏，左栏顶部「传统方式」，下方画一个老式图示；
右栏顶部「新方式」，下方画一个现代图示，
两栏底部有一个总结箭头从左指向右。

[布局描述]：左右两栏等宽，中间留白 15%，
左栏整体用灰蓝色调（表示过时），右栏用暖橙色调（表示推荐），
总结箭头跨越两栏。
```

### 4. Pyramid / Stack（层级堆叠）
适合：分层架构、金字塔结构。
```
[主要视觉描述]：画面中央一个从下到上的金字塔（或分层堆叠），
每层标注一个架构层名称，
每层右侧延伸出 1-2 个该层的代表图标。

[布局描述]：竖向堆叠 4 层，最底层最宽用深青绿色，
向上逐层收窄，颜色从深青绿过渡到暖橙，
最顶层最窄用红色表示关键层。
```

### 5. Grid / Matrix（矩阵分类）
适合：2x2 / 3x3 分类图。
```
[主要视觉描述]：2x2 矩阵，每个象限代表一个类别，
每个象限内有一个代表性图标和类别名称。

[布局描述]：2 行 2 列均分画布，象限间用细线分隔，
四个象限色调分别为暖橙/深青绿/柔蓝/珊瑚红，
中心交点有一个小装饰圆点。
```

## 常见失败与修复

- **生成太杂乱** → 在结尾追加"极简风格，减少元素"
- **插画像简笔画** → 强调"专业扁平矢量插画，精致细节"
- **颜色跑偏** → 用精确十六进制（`#e8943a`）代替"橙色"等模糊词
- **文字错别字** → 在图内文字用「」包起来，例如「智能体」
- **比例不对** → 确认末尾有 `--ar 9:16`，或 image_gen 调用时 `size="1024x1536"`
