# 演讲稿写作指南

演讲稿决定视频质量。必须反复润色，不能草率。

## 1. 提炼：从原文到核心骨架

原文通常信息量远大于视频容量。第一步是**做减法**，不是搬运原文，而是提炼。

**三步提炼法：**
1. **读完原文，用一句话概括全文核心主张**（概括不出来说明还没理解透）
2. **列出支撑这个主张的 3-5 个关键论点**（这些论点就是场景候选）
3. **砍掉观众不需要知道的细节**——视频不是文档，观众只需要"啊原来是这样"的顿悟感

**提炼公式：**
```
原文 1000+ 字 -> 核心主张（1句话）+ 关键论点（3-5个）-> 7-8 个场景
```

## 2. 结构：五种经典叙事骨架

根据内容类型选一种骨架，不要凭感觉随意编排：

| 骨架类型 | 适用场景 | 结构 |
|---------|---------|------|
| **揭秘型** | 源码分析、内幕爆料 | 事件引爆 → 全景概览 → 逐层深入 → 总结升华 |
| **问题-方案型** | 技术痛点、最佳实践 | 痛点共鸣 → 原因分析 → 解决方案 → 效果对比 |
| **概念拆解型** | 新技术/新概念科普 | 类比引入 → 定义拆解 → 组成部分 → 意义/趋势 |
| **对比型** | A vs B、新旧对比 | 展示旧方式 → 暴露问题 → 引入新方式 → 对比总结 |
| **清单型** | Top N、最佳实践列表 | 开场定调 → 逐条展开 → 总结串联 |

## 3. 开场：3 秒定生死

竖屏视频前 3 秒决定观众划不划走。

| 模板 | 公式 | 示例 |
|------|------|------|
| **事件型** | [最近发生的事] + [我的发现] | "前段时间 Claude Code 源码泄露了,我花了两天拆开看" |
| **反常识型** | [你以为的] + [其实是] | "你以为 Claude 只会写代码? 其实你只用了它 20%" |
| **痛点型** | [你遇到过 X 吧] + [原因在这里] | "你的 Agent 上线老是崩? 大概率是缺了一层东西" |

**开场禁忌：**
- ❌ "大家好，今天我来给大家讲一下..."
- ❌ 从定义开始讲（"XXX 是一种..."）
- ❌ 铺垫超过 15 秒

## 4. 收尾：给一个带走的金句

观众看完视频能记住的，往往只有最后一句话。

**三要素：**
1. **回扣主题**：把开头抛出的问题/悬念收回来
2. **升华观点**：从具体细节拉到更大的认知
3. **一句话金句**：可以被观众直接转述给别人的那种

**好 vs 坏：**
- ❌ "以上就是今天的内容,希望对大家有帮助"（废话）
- ❌ "试试这 39 个工具,你会发现全新的世界"（空洞）
- ✅ "生产级的 Agent 不是模型多聪明的问题,是工程纪律有多严的问题. 这才是 Anthropic 真正的护城河."

## 5. 口语化：老友聊天风格

整体像**跟朋友喝咖啡聊技术**——自然随意但不夸张，有个人观点，偶尔吐槽，不用网络流行语。

**风格要点：**
- ✅ 自然、平静、有观点："我看了下，挺有意思的"
- ✅ 偶尔反问引导思考："你猜哪类工具最多?"
- ✅ 个人判断和感受："让我意外的是..."
- ❌ 不用夸张语气词："好家伙""炸锅了""牛逼""绝了"
- ❌ 不用自媒体套话："建议收藏""99% 的人不知道"
- ❌ 不刻意制造情绪："震惊""太猛了""颠覆认知"

**翻译对照：**

| 书面语 | 老友聊天风格 |
|--------|------------|
| 该工具支持 5 种内容类型 | 这个工具能读 5 种东西 |
| 源码中存在一个优化机制 | 这里有个挺聪明的设计 |
| 此设计防止 AI 盲目覆盖文件 | 这样就不容易让 AI 把你的代码搞坏了 |
| 该框架具有高度可扩展性 | 扩展起来很方便 |

**口语化检验法：** 把稿子念出来，想象你在跟技术朋友聊天。如果对方会觉得你在"演讲"或"卖课"，就还不对。

## 6. 场景过渡：缝合不是跳切

**过渡公式：[上一场景的总结/钩子] + [引出下一场景]**

| 类型 | 公式 | 示例 |
|------|------|------|
| **递进型** | "X 搞定了，那 Y 呢？" | "文件搞定了,搜索和执行命令呢? 坑更多." |
| **揭示型** | "看完 X，你会发现 Y" | "看完这些工具,会发现藏了四个设计原则." |
| **转折型** | "但 X 只是表面，真正的 Z 是..." | "但这只是冰山一角,真正让我震撼的是..." |
| **聚焦型** | "全局看完了，挑 X 拆开看" | "七大类里用得最多的是文件系统.拆开看." |

## 7. 自检清单

写完稿子逐条检查：

- [ ] 一句话能概括全片主张吗？
- [ ] 开场 3 秒有没有钩子？念前两句会不会划走？
- [ ] 每个场景只讲 1 个核心点？（塞了 2 个以上就拆开）
- [ ] 场景之间有过渡语吗？
- [ ] 收尾有金句吗？能被别人转述吗？
- [ ] 念出来像聊天还是读稿？
- [ ] **每场景 140-170 字？** 7-8 场景，总计 1000-1200 字
- [ ] **中文标点已替换为英文？** `，->,` `。->.` `？->?`

## 技术约束汇总

- 每场景 **140-170 字**（对应 22-26 秒语音，腾讯云 TTS 专业梓欣 602005 + 1.1 倍速，实测约 6.5 字/秒）
- 中文标点用英文标点替代（TTS 要求）
- 7-8 个场景，总字数 1000-1200 字，总时长约 3 分钟
